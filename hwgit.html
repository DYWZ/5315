What is GIT? Why do we need source code Version Control System?

Git is an open source project initially developed in 2005 by Linus Torvalds, the famous creator of the Linux operating system kernel. Commercial and open source software projects rely on Git for version control. With VCS, everyone on the team can work on any file individually at any time. VCS will later allow you to merge all changes into a common version. VCS enhances team collaboration. Even though the version control system recognizes that there is only one project, VCS packages all past versions and variants. When problems arise, people can find the source of the problem by having information about earlier versions available at any time through VCS. VCS also can restore previous versions. Each time a new version of a project is saved, VCS asks people to provide a short description of what has changed, which documents the evolution of different versions. A side effect of using a distributed VCS like Git is that it can be used as a backup. If someone's central server fails (and your backup drive fails), just restore one of your teammates' local Git repositories.

Describe following terms: 1) local repo 2) remote repo 3) how to use GIT in Team Project - multiple developers work on the same project at the same time

1) The local repo is on one's computer and has all the files and their commit history, enabling full diffs, history review, and committing when offline. A local repository has a a directory associated with it. 2) Remote repositories are hosted on a server that is accessible for all team members. Remote repo is like a "file server" that can be used to exchange data with team members. 3) People can use Git to check out a new branch, work on it, then merge the changes back into the master branch in a single, curated commit. Git can keep one's master branch nice stays and neat by merging a single commit with all the changes.
